Diretório da biblioteca da ST (libstm32.a)
------------------------------------------

Pré-requisitos:

  - CodeSourcery G++ Lite (target OS: EABI) - ferramentas de desenvolvimento.

Conteúdo:
  - src             : diretório de códigos-fontes. 
                      Obs.: Basta incluir um arquivo ".c" neste diretório para 
                      que ele seja tratado como um novo código-fonte da
                      biblioteca. 
  - inc             : diretório de arquivos de cabeçalho.
                      Obs.: Para incluí-los nos códigos-fontes da biblioteca 
											ou do projeto, basta usar um " #include "cabelho.h" ".
											Essa característica é possível pelo uso da opção "-I" 
											utilizada para compilar a biblioteca e o projeto, a qual
											indica o diretório "inc" com diretório de procura por 
											arquivos de cabelho.
											
  - template        : diretório de templates.
                      Obs.: para utilizar os templates, prefira copiá-los e 
                            renomeá-los a copiar seu conteúdo, pois assim é 
                            mantido o tipo de codificação do arquivo, 
                            fundamental para o uso da ferramenta Doxygen.
  - CLEAN.cmd       : elimina todos os arquivos gerados durante a compilação 
                      da biblioteca.
  - LEIA-ME.txt     : este arquivo.
  - MAKE_ALL.cmd    : compila somente o que foi alterado no projeto, gerando 
                      um novo arquivo ".a".
  - Makefile        : arquivo utilizado para gerenciar toda a biblioteca, 
                      possuindo as regras e comandos necessários para compilar 
 										  e gravar o projeto.
                      Obs.: digite "cs-make help" de dentro deste diretório 
                      para obter ajuda sobre como utilizá-lo.
  - REBUILD_ALL.cmd : limpa e recompila todos os arquivos da biblioteca, 
	                    gerando um novo arquivo ".hex".

NOTA:

  - Por possuir um arquivo "Makefile", o qual concentra todas as regras e 
	  comandos necessários para a compilação da biblioteca, este projeto pode 
		ser utilizado por qualquer IDE que possua suporte à execução de
		ferramentas externas.
		Vide os arquivos (.cmd) ou "cs-make help" para obter mais informações de 
		como utilizar o arquivo Makefile deste projeto em seu IDE preferido.
		Exemplos de IDE: Programmer's Notepad e Eclipse.
